# 路由生成器 `routerGenerator.js` 使用指南

本指南详细介绍了如何使用 `routerGenerator.js` 工具来自动化您的 Vue 项目路由配置。

## 核心理念

此工具的核心是通过扫描您在 `src/views` 目录下的文件结构，自动创建路由规则。您不再需要手动在 `router/index.js` 中逐一添加路由。这大大简化了路由管理，提高了开发效率，并降低了出错的可能性。

## 文件结构约定

为了让路由生成器正常工作，您需要遵循以下文件结构约定：

-   每一个页面（即一个独立的路由）都应该在 `src/views` 目录下拥有一个自己的文件夹。
-   页面组件应命名为 `index.vue`。
-   页面的元数据（meta information）和路由配置应放在同级目录的 `page.js` 文件中。

**示例结构:**

```
src/
├── views/
│   ├── home/                # 对应 /home 路由
│   │   ├── index.vue      # 页面组件
│   │   └── page.js        # 页面路由配置
│   │
│   ├── login/               # 对应 /login 路由
│   │   ├── index.vue
│   │   └── page.js
│   │
│   └── user/
│       ├── profile/         # 对应 /user/profile 路由
│       │   ├── index.vue
│       │   └── page.js
...
```

## `page.js` 配置文件详解

`page.js` 文件通过其导出的 `layout` 属性来决定页面使用哪个父组件（布局），或者完全不使用布局。

### 1. 使用特定布局的页面

要让一个页面使用某个布局，您需要在其 `page.js` 文件中设置 `layout` 属性。这个属性的值是一个**字符串名称**，它必须与您在 `router/index.js` 中配置的布局名称相对应。

-   **为什么是 `'default'`？** `'default'` 是我们约定的默认主布局的名称。您可以将其视为“网站的主要框架”。
-   **可以是其他值吗？** 当然可以！如果您有另一个布局，比如一个用于管理后台的 `AdminLayout.vue`，您可以将其命名为 `'admin'`。然后在需要使用该布局的页面的 `page.js` 中写 `layout: 'admin'`。

**示例 1: 使用默认主布局 (`src/views/home/page.js`)**

```javascript
export default {
  title: '首页',
  requiresAuth: true,
  layout: 'default', // **关键**: 指定此页面使用名为 'default' 的布局
};
```

**示例 2: 使用后台管理布局 (`src/views/admin/dashboard/page.js`)**

```javascript
export default {
  title: '管理仪表盘',
  requiresAuth: true,
  layout: 'admin', // **关键**: 指定此页面使用名为 'admin' 的布局
};
```

### 2. 独立页面（无布局）

对于像登录页、注册页或 404 错误页这样不需要任何父布局的页面，只需在 `page.js` 的导出对象中 **省略 `layout` 属性** 即可。生成器会将其识别为顶级的、独立的路由。

**示例: `src/views/login/page.js`**

```javascript
export default {
  title: '登录',
  requiresAuth: false,
  // 此处没有 layout 属性，因此它将成为一个独立的顶级路由
};
```

## `router/index.js` 中的设置（支持多布局）

为了支持多布局，您需要在调用生成器时传入一个 `layoutComponents` 对象，它是一个**布局名称到布局组件的映射表**。

```javascript
import { createRouter, createWebHistory } from 'vue-router';
import { createRoutesGenerator } from '../utils/routerGenerator';

// 导入您的所有布局组件
import MainLayout from '../components/layout/AdminLayout.vue';
import AdminLayout from '../components/layout/AdminLayout.vue'; // 假设您有另一个布局

// 1. 静态加载页面和组件 (保持不变)
const pages = import.meta.glob('../views/**/page.js', { eager: true, import: 'default' });
const components = import.meta.glob('../views/**/index.vue');

// 2. 创建生成器实例 (保持不变)
const generateRoutes = createRoutesGenerator({
    pages,
    components,
    basePath: '../views',
});

// 3. 调用生成器，并提供布局映射表
const routes = generateRoutes({
    layoutComponents: {
        // 将 'default' 名称映射到 MainLayout 组件
        'default': MainLayout,
        // 将 'admin' 名称映射到 AdminLayout 组件
        'admin': AdminLayout,
    },
    // （可选）排除某些路由
    exclude: ['/temp'],
});

// 创建 Vue Router 实例 (保持不变)
const router = createRouter({
    history: createWebHistory(),
    routes,
});

export default router;
```

## 总结

通过以上配置，您可以实现一个高度自动化、可配置且易于维护的路由系统。当您需要添加新页面时，只需在 `src/views` 目录下创建对应的文件夹和文件，路由系统会自动更新，无需任何额外的手动配置。
